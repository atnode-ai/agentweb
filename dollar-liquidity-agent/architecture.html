Agent: Matthew

Architecture

What Matthew Does

Matthew is a scheduled monitoring agent built on Hyperagent. Every Friday morning it pulls data from five primary sources, runs nine calibrated alert thresholds, scans Tier A/B news wires for corroborating evidence, and posts a structured status report — GREEN, AMBER, or RED — to a private Slack channel. The entire cycle runs unattended in under two minutes.

Data Sources

Each cycle queries four federal data APIs and three Japanese government sources, producing roughly 30 individual data points per run:

H.4.1 via FRED
SWPT, WMTSEC1, WSEFINT1 — weekly swap balances and foreign custody of Treasuries. Six-week rolling trend with week-on-week and baseline deltas.
NY Fed Markets API
Standing swap-line operations and FIMA Repo Facility draws. Checked for non-zero activity by non-baseline counterparties.
TIC SLT (Treasury)
Monthly major foreign holders of U.S. Treasuries. Country-level tracking for Japan, China, Saudi Arabia, UAE, and 20 others. Two-month publication lag.
Japan MoF / BoJ
Monthly foreign reserves (total, FX, gold, SDR), BoJ balance sheet (FX assets, government securities), confirmed FX intervention operations, and USD/JPY spot rate.

News scanning covers Tier A sources (Federal Reserve and U.S. Treasury press releases, FOMC minutes, central-bank governor statements) and Tier B sources (Bloomberg, Reuters, Financial Times, Wall Street Journal, CNBC, Nikkei). Analyst opinion and Substack commentary are noted for context but cannot trigger an alert alone.

Nine Alert Thresholds

Thresholds are calibrated to avoid false positives. Matthew is designed to be hard to startle — its value is that when it does alert, the signal is real. The framework distinguishes strong stress signals (multi-country swap-line cluster), medium signals (official approval or facility usage), and weak/noise signals (single-country informal inquiry).

Execution Architecture

The agent runs on Hyperagent's scheduled invocation system. A single Python runner script (run_weekly_cycle.py) orchestrates all four data fetches, computes deltas against stored baselines, evaluates all nine thresholds, and outputs structured JSON plus a pre-formatted Slack message. The agent layer then scans news sources, applies editorial judgment on status calibration, posts to Slack, updates the project document, and pushes the dashboard to GitHub Pages.

API credentials (FRED, GitHub) are stored as skill-level secrets, injected at runtime, and never appear in conversation history or output. The agent maintains a rolling eight-week window of research findings, with older entries rotated to an archive document to keep the working context lean.

All scripts, thresholds, and baseline values are version-controlled as a reusable Hyperagent skill, so the monitoring logic can be updated, audited, and rolled back independently of the agent's conversational prompt.
